# Security Policy

Please do not report vulnerabilities in public issues.

Until a dedicated security contact is published, report sensitive issues to the
project maintainer privately with a clear reproduction path, affected version,
and suggested mitigation. Do not include API keys, session cookies, databases,
or customer files in reports.

RelayDesk is intended for trusted internal networks, but it treats browsers,
uploads, Runtime responses, and Runtime URLs as untrusted input.
